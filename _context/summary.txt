Brain MVP command layer implementation summary

Verified saved on 2026-07-01.

Implemented and tightened the repo-only command layer for the locked Brain MVP workflows:

- good morning
- replan day
- update brain

Changed command behavior:

- goodMorning and replanDay now pass active goals and the latest BrainUpdateReport into the planning context.
- Planning now filters inactive projects using both project status and executionState.
- Ignored project states now include blocked, completed, abandoned, archived, and ready_for_production.
- Closed task filtering now includes closed, complete, completed, dismissed, archived, and converted.
- updateBrain now snapshots DayPlans before running and verifies DayPlans were not created, updated, deleted, restarted, or archived.
- Command scripts remain executable from npm scripts and now expose importable runner functions for tests.
