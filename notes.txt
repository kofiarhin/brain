# Brain Vision Handoff: Autonomous Personal Operating System

## Vision

Transform **Brain** from a web application into an **autonomous personal operating system**.

The web UI becomes one interface to Brain, but the primary interaction model is through **Codex Desktop Automations**, allowing Brain to proactively execute routines, gather information, and produce outputs on a schedule.

Brain owns the intelligence and workflows.

Codex Desktop provides the scheduling and execution engine.

---

# Core Architecture

```
                     Codex Desktop
                   (Automations)
                          │
         ┌────────────────┼────────────────┐
         │                │                │
         ▼                ▼                ▼
   Morning Job      Midday Job      Evening Job
         │                │                │
         └────────────────┼────────────────┘
                          │
                          ▼
                     Brain Core
                          │
     ┌──────────────┬──────────────┬──────────────┐
     │              │              │              │
 Calendar       Gmail         GitHub        Projects
     │              │              │              │
     └──────────────┴──────────────┴──────────────┘
                          │
                          ▼
                    AI Planning Engine
                          │
                          ▼
                 Reports / Tasks / Dashboard
```

The Brain UI simply becomes another client of the Brain Core.

---

# Long-Term Goal

Brain should work even when the user never opens the website.

Instead, Codex automatically runs workflows throughout the day and keeps Brain up to date.

The user wakes up to completed work instead of a dashboard waiting for input.

# Design Principle

The current "Good Morning" button should **not** contain the planning logic.

Instead:

```
Brain UI
      │
      ▼
Brain Service
      │
Planning Pipeline
```

Codex Automations should invoke the exact same service.

There should only be one implementation of each workflow.

---

# End Goal

Brain evolves from a reactive application into a proactive operating system.

Instead of opening Brain to ask, "What should I do today?", the system continuously works in the background—reviewing your projects, GitHub repositories, calendar, emails, notes, and tasks—and delivers actionable outputs throughout the day. Codex Desktop serves as the autonomous execution layer, while Brain remains the centralized intelligence and workflow engine. This creates a scalable architecture where new workflows can be added simply by defining a Brain service and attaching a Codex Automation schedule.
